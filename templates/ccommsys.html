Communication systems enable people to send and receive data and information. With the introduction of the telephone, radio, television and internet, our dependance on communication systems have greatly increased.
All communication systems have five basic components

  1. The data source produces the data to be sent
  2. The transmitter encodes data into a signal suitable for a transmission medium.
  3. The transmission medium is a device which allows the signal to be transmitted to the destination.
  4. The receiver decodes the signal back into the original data, or an approximation of the data.
  5. The destination is the receiver of the information.

[Five components of communication systems] Davis, S. Information Processes and Technology - The HSC Course. P229 Published 2004.

A protocol is a set of rules that regulates transfer of data between computers.
Handshaking is an agreement about which protocol to use, made by devices that communicate with each other.
Bandwidth is the capacity of the transmission medium. A cable with high bandwidth can transfer more data than one with low bandwidth. An example of a high bandwidth transmission medium is fibre-optic.
Speed of data transfer is measured by the number of bits per second or baud rate:
  • Bits per second is the maximum number of bits that can be transmitted in one second.
  • Baud rate is the maximum number of data symbols or electrical signals that can be transmitted in one second.
Error checking methods
There are three methods commonly used for error checking:
  • Parity checking involves checking for errors using what is called a parity bit.
  • Odd parity bit checks if the number of ones in the transmitted bits is odd. If the number of ones are even, then the parity bit is set to 1 to meet the criteria of odd parity.
  • Even parity bit checks if the number of ones in the transmitted bit is even, and the same principle of odd parity applies here. If there are an odd emount of ones, the parity bit is set to 1.
  • Checksum involves the receiver checking for the total number of bits and comparing them to what was transmitted.
  • Cyclic redundancy check is a method of checking for errors by using a divison process. The data is divided into predetermined lengths and divded by a fixed divisor.
